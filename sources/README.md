# Canonical source tree

Every supported platform has lifecycle directories. YouTube, academic, social, and legacy evidence records use Markdown with YAML front matter. The web/media lanes (`blogs`, `podcasts`, `books`, `conferences`, `case-studies`, and `github`) use JSON records with bounded evidence arrays. Record format and path select the validator; `schema_version` alone does not.

Accepted, rejected, and blocked records all remain in canonical accounting, but only accepted relevant records contribute to validated-source totals. Artifact level, rights status, and provenance are preserved in deterministic metadata and statistics.
