# Obsidian is the IDE.

## Complete source text

Obsidian is the IDE.
The LLM is the programmer.
The wiki is the codebase.

Great! Over the past two months, I’ve been building a system around my personal X bookmarks and social content. Today, I fed @karpathy’s ideas into the agent system and let it upgrade itself.

Any agent can load this repo to get a full self-IP agent system:
https://github.com/tagai-dao/self-ip-agency

It consists of 3 agents: main, bookmarker, and trader.

Here’s the core loop:

Data ingestion
X posts + bookmarks → summaries → agent memory docs → Obsidian

Autonomous social
Summaries → autonomous social workflow → the agent publishes and interacts on its own

Personal alignment
My personal Twitter account interacts with the agent’s content → personal alignment score

Community evaluation
Community agents interact with the agent’s content → community evaluation score

TAS_social
Personal alignment + community evaluation → TAS_social

Autonomous evolution
Each heartbeat cycle, the agent tries different strategies to optimize TAS_social
→ compare whether TAS_social went up or down
→ reinforce what works, discard what doesn’t
→ evolve autonomously

The trader agent runs in a similar way, but focused on trading.

The main agent orchestrates both the bookmarker and trader agents, and coordinates their actions to improve overall returns.

Together, this produces TAS: a weighted combination of TAS_social and TAS_trade.

TAS is the core metric driving the evolution of the self-IP 3-agent system.
It was autonomously created by the agent itself, inspired by @karpathy’s Autoresearch experiment.

This system has already been running autonomously for over a month and is still evolving.
